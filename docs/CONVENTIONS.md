# Code Conventions & Patterns

## Dart/Flutter Patterns

### Riverpod (Manual Style - No Code Gen)

Providers are declared manually without annotations:

```dart
// Repository providers - throw error, overridden in main.dart
final studyPlanRepoProvider = Provider<StudyPlanRepository>((ref) {
  throw UnimplementedError('Must be overridden in main');
});

// StateNotifier for mutable state
final settingsProvider = StateNotifierProvider<SettingsNotifier, AppSettings>((ref) {
  return SettingsNotifier(ref);
});

// Derived providers for computed values
final overallProgressProvider = Provider<double>((ref) {
  final plans = ref.watch(weekPlansProvider);
  final completed = ref.watch(completedTaskIdsProvider);
  // ... compute and return
});

// Parameterized derived providers
final weekProgressProvider = Provider.family<double, int>((ref, weekNumber) {
  // ... compute progress for specific week
});
```

### Hive Models (Manual TypeAdapters)

Models use `@HiveType` and `@HiveField` annotations but adapters are **hand-written** in `.g.dart` files (not generated by build_runner). This was a deliberate choice.

```dart
// Model definition
@HiveType(typeId: 0)
class WeekPlan extends HiveObject {
  @HiveField(0)
  final int weekNumber;
  // ...
}

// Adapter in .g.dart - MANUALLY WRITTEN
class WeekPlanAdapter extends TypeAdapter<WeekPlan> {
  @override
  final int typeId = 0;
  // ...
}
```

TypeId assignments:
- 0: WeekPlan
- 1: TaskItem
- 2: ProgressEntry
- 3: AppSettings
- 4: Reflection

### Repository Pattern

Repositories are thin wrappers around Hive boxes:

```dart
class SomeRepository {
  static const _boxName = 'box_name';
  late Box<Model> _box;

  Future<void> init() async {
    _box = await Hive.openBox<Model>(_boxName);
  }
  // CRUD methods
}
```

All repos are initialized in `main.dart` before `runApp()` and injected via `ProviderScope.overrides`.

### Animation Patterns

- **Staggered slide-in**: `_SlideIn` widget with `AnimationController` + `Interval` for delay
- **Animated counters**: `TweenAnimationBuilder<double>` wrapping text that displays `(value * 100).round()%`
- **Scale bounce**: `AnimationController` with `Curves.easeOutBack` on nav items
- **Glassmorphism**: `BackdropFilter` with `ImageFilter.blur(sigmaX: 30, sigmaY: 30)` + semi-transparent container

### Custom AnimatedBuilder

A local `AnimatedBuilder` class exists in both `dashboard_screen.dart` and `animated_nav_bar.dart` (extends `AnimatedWidget`). This was created to work around naming/parameter issues with Flutter's built-in `AnimatedBuilder`.

## iOS/Swift Patterns

### WidgetKit Extension

- `TimelineProvider` pattern for widget data
- Shared data via App Group UserDefaults
- `@main struct CloudStudyWidgets: WidgetBundle` registers all widget types
- Each widget family has its own SwiftUI view struct
- Phase colors duplicated in Swift to match Flutter side

### MethodChannel

Channel name: `com.cloudstudy/widgets`
- Single method: `updateWidgets`
- Args: `{ appGroup: String, data: String (JSON) }`
- Called from Flutter whenever progress changes or on app startup

## File Naming

- Models: `snake_case.dart` with `.g.dart` companion for adapters
- Screens: `<feature>_screen.dart` in `screens/<feature>/` directory
- Providers: `<domain>_provider.dart`
- Services: `<domain>_service.dart`
- Widgets: `snake_case.dart` in `shared/widgets/`
- Constants: in `shared/constants/`
- Extensions: in `shared/extensions/`

## Styling Conventions

- `GoogleFonts.jetBrainsMono(...)` for all headings, labels, badges, monospace UI
- `GoogleFonts.ibmPlexSansTextTheme(base)` for body text (set at theme level)
- Phase colors use a `PhaseColors` class with `bg`, `border`, `text` fields
- `withValues(alpha: x)` instead of deprecated `withOpacity(x)` for colors
- Border radius: 12-20px for cards/containers, 4-6px for badges
- Consistent padding: 16px standard, 24px for hero cards

### copyWith Pattern for Hive Models

Since Hive returns the same object reference on reads, Riverpod StateNotifiers won't detect changes. Use `copyWith()` to create new instances:

```dart
// AppSettings has a copyWith() method
state = s.copyWith(); // creates a new instance, triggers StateNotifier

// For adding fields: add to constructor, copyWith, and TypeAdapter (.g.dart)
```
